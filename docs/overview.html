Welcome to the API reference for VoxelGameEngine — a small, modular voxel game engine written in Java 21 on top of LWJGL and OpenGL. If you have ever wanted to see how a Minecraft-style world is loaded, meshed, lit, and drawn without wading through a monolith, you are in the right place.

What this engine is

The project is split into focused modules so that each concern can be read, tested, and reasoned about on its own. Nothing here is magic: a world is a grid of chunks, a chunk is a cube of blocks, and the renderer turns the visible faces of those blocks into triangles on your screen. The interesting parts are the details — greedy-ish meshing, frustum and occlusion culling, cascaded shadow maps, threaded chunk generation, and a lightweight physics/collision pass.

How the modules fit together

Dependencies flow in one direction, from the concrete game down to the shared core. This keeps the lower layers reusable and free of surprises.

  • engine-core — the shared foundation: math types ({@code Vec3i}, {@code Vec3f}, {@code AABB}), configuration, logging, noise, and fast primitive collections. Everything depends on this; it depends on nothing in the engine.
  • engine-platform — the thin layer over GLFW that opens a window and reads keyboard and mouse input.
  • engine-world — the data model of the world: blocks, the block registry, chunks, coordinate math, and terrain generation (including the threaded generation pipeline).
  • engine-renderer — everything OpenGL: shaders, textures, chunk meshing, culling, lighting, and the immediate-mode-style UI toolkit used for menus and the HUD.
  • engine-physics — gravity, Euler integration, collision resolution against the voxel grid, and raycasting (used for picking the block you are looking at).
  • engine-assets — loading image data off disk into a form the renderer can upload to the GPU.
  • game — gameplay glue: the player and the controller that wires input, camera, physics, and world edits together.
  • launcher — the entry point. It reads the config, builds the {@code GameLoop}, and runs it.

Where to start reading

If you are new to the codebase, a good tour is:

  1. {@code de.heger.voxelengine.launcher.GameLoop} — the heartbeat that ties input, world, physics, and rendering together each frame.
  2. {@code de.heger.voxelengine.world.chunk.Chunk} and {@code ChunkManager} — how the world is stored and streamed.
  3. {@code de.heger.voxelengine.renderer.Renderer} and {@code mesh.ChunkMeshBuilder} — how voxels become pixels.
  4. {@code de.heger.voxelengine.core.math.Vec3i} — the humble integer vector that block and chunk coordinates are built on.

Every package has its own short overview describing what lives there and why. Follow the Packages list to dive in.
