API reference for VoxelGameEngine, a small modular voxel game engine written in Java 21 on top of LWJGL and OpenGL.

What this engine is

The project is split into modules, one per concern, so that each can be read and tested on its own. A world is a grid of chunks, a chunk is a cube of blocks, and the renderer turns the visible faces of those blocks into triangles on your screen. On top of that: greedy-ish meshing, frustum and occlusion culling, cascaded shadow maps, threaded chunk generation, and a lightweight physics/collision pass.

How the modules fit together

Dependencies flow in one direction, from the concrete game down to the shared core.

  • engine-core: the shared foundation. Math types ({@code Vec3i}, {@code Vec3f}, {@code AABB}), configuration, logging, noise, and fast primitive collections. Everything depends on this; it depends on nothing in the engine.
  • engine-platform: a thin layer over GLFW that opens a window and reads keyboard and mouse input.
  • engine-world: the data model of the world. Blocks, the block registry, chunks, coordinate math, and terrain generation (including the threaded generation pipeline).
  • engine-renderer: everything OpenGL. Shaders, textures, chunk meshing, culling, lighting, and the UI toolkit used for menus and the HUD.
  • engine-physics: gravity, Euler integration, collision resolution against the voxel grid, and raycasting (used for picking the block you are looking at).
  • engine-assets: loading image data off disk into a form the renderer can upload to the GPU.
  • game: gameplay glue. The player and the controller that wires input, camera, physics, and world edits together.
  • launcher: the entry point. Reads the config, builds the {@code GameLoop}, and runs it.

Where to start reading

  1. {@code de.heger.voxelengine.launcher.GameLoop}: ties input, world, physics, and rendering together each frame.
  2. {@code de.heger.voxelengine.world.chunk.Chunk} and {@code ChunkManager}: how the world is stored and streamed.
  3. {@code de.heger.voxelengine.renderer.Renderer} and {@code mesh.ChunkMeshBuilder}: how voxels become pixels.
  4. {@code de.heger.voxelengine.core.math.Vec3i}: the integer vector that block and chunk coordinates are built on.
